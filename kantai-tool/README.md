# 艦隊管理ツール (Kantai Tool)

このプロジェクトは Create React App で作成された、艦これの艦隊編成管理ツール（フロントエンド）です。

## セットアップ (データ準備)

このアプリケーションの動作には艦娘マスターデータ (`shipMaster.json`) が必要です。
サイズが大きいため、Git管理からは除外されています。

1. `shipMaster.json` を入手（または生成）してください。
2. `public/` ディレクトリに配置してください (`kantai-tool/public/shipMaster.json`)。

## 使い方

1. **バックエンドの起動**: 別途 `kanmusu-server` を起動しておく必要があります（ポート3002）。
2. **ログイン**: 任意のユーザー名を入力してログインします（初回は自動登録）。
3. **艦隊編成**:
    - **ドラッグ＆ドロップ**: 艦娘一覧から艦隊スロットへドラッグして編成します。
    - **7隻目**: 「+」ボタンで遊撃部隊（7隻編成）に対応します。
    - **連合艦隊**: 「連合艦隊表示」にチェックを入れると、第1・第2艦隊を並べて表示します。
4. **データ保存**: 「💾 編成を保存」ボタンでサーバーに保存されます。
5. **特効作成**: 「⚡ 特効作成」モードで、イベント海域ごとの特効艦グループを作成・保存できます。

## 利用可能なスクリプト

プロジェクトディレクトリ内で以下のコマンドを実行できます:

### `npm start`

開発モードでアプリを起動します。\
ブラウザで http://localhost:3000 を開いて確認してください。

編集するとページがリロードされます。\
コンソールにはLintエラーも表示されます。

### `npm test`

対話型ウォッチモードでテストランナーを起動します。\
詳細は running tests を参照してください。

### `npm run build`

本番環境用にアプリをビルドし、`build` フォルダに出力します。\
Reactを本番モードでバンドルし、パフォーマンスのために最適化します。

ビルドは最小化され、ファイル名にはハッシュが含まれます。\
これでデプロイの準備が整いました！

詳細は deployment を参照してください。

### `npm run eject`

**注意: これは不可逆的な操作です。一度 `eject` すると元に戻せません！**

ビルドツールや設定に満足できない場合、いつでも `eject` できます。このコマンドは、プロジェクトから単一のビルド依存関係を削除します。

代わりに、すべての設定ファイルと推移的な依存関係（webpack, Babel, ESLintなど）をプロジェクトにコピーし、完全に制御できるようにします。`eject` 以外のコマンドは引き続き機能しますが、コピーされたスクリプトを指すようになります。ここからは自分自身で設定を管理することになります。
